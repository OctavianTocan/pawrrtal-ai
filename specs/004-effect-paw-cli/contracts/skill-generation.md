# Skill Generation Contract

The new CLI skills are generated by `packages/ci/skill-gen/`.

## Generated Outputs

| Skill | Output | Purpose |
| --- | --- | --- |
| `paw` | `.agent/skills/paw/SKILL.md` | Teaches agents how to use the supported Paw CLI. |
| `domain-cli` | `.agent/skills/domain-cli/SKILL.md` | Teaches agents how to change, extend, test, and review the CLI package. |

## Sources

`skill-gen` keeps its existing source marker scan and also accepts dynamic fragments from `packages/paw-cli`.

Required dynamic fragment source:

```text
packages/paw-cli/src/Skills/Fragments.ts
```

That source reads the canonical command descriptors and emits fragments compatible with `skill-gen` merge/output rules.

## Required Public Commands

No new public generator command is introduced.

```bash
bun run skill-gen:generate
bun run skill-gen:check
bun run skill-gen:e2e-test
```

## Drift Rules

`bun run skill-gen:check` must fail when:

- `.agent/skills/paw/SKILL.md` is missing.
- `.agent/skills/domain-cli/SKILL.md` is missing.
- Either generated skill differs from command descriptors or domain guidance.
- Either generated skill cites removed Python CLI source files.

## Generated `paw` Skill Minimum Content

- Installation and launcher commands.
- Root command usage.
- Built-in commands: `doctor`, `context`, `completions`.
- Output modes and stream contract.
- Exit code table.
- Config and environment variables.
- Current command groups.
- Verification commands.
- Pitfalls and stale/cache/degraded state behavior.

## Generated `domain-cli` Skill Minimum Content

- Package layout for `packages/paw-cli`.
- How command descriptors become Effect commands.
- How command descriptors feed generated skills.
- How to add a feature-owned command group.
- Output/error/config/input-source conventions.
- Testing expectations.
- Required gates.

## One-Time Migration Cleanup

The old Python CLI cleanup is an implementation task for this feature, not permanent `domain-cli` skill content. During the first implementation pass:

- Remove generated fragments that cite `backend/app/cli/paw/`.
- Remove or retire `paw-extend` if it only teaches old Python CLI maintenance.
- Keep `domain-cli` focused on the new package layout and extension workflow after the cleanup is complete.

## Compatibility With Existing `skill-gen`

- Static `//<skill-gen>` fragments keep working.
- Dynamic fragments are normalized into the same parsed fragment model.
- Output still includes `<!-- AUTO-GENERATED by skill-gen -- DO NOT EDIT -->`.
- Hand-written skills without the generated header are not overwritten.
