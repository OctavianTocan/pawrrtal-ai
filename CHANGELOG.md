# Changelog

## Unreleased

### Fixed
- **CI: all 16 checks green from a clean branch off `development`.** Fixed 206 React Doctor warnings (unused exports, missing aria-labels, non-semantic HTML, circular dependencies, file splits for oversized components), flattened backend nesting-depth violations, fixed MDX build failures from angle-bracket URLs and unquoted YAML frontmatter, fixed backend pytest key-set assertion to use canonical `OVERRIDABLE_KEYS`, removed stale Playwright assertion for deleted "Preferences" UI, added `WORKSPACE_BASE_DIR` / `generate:docs` / Node setup to Stagehand workflow, added `aria-label` to all settings Switch toggles, and namespaced Stagehand API keys (`STAGEHAND_GOOGLE_API_KEY` etc.) so the backend's `GOOGLE_API_KEY` doesn't leak into the E2E LLM fixture.
- **Telegram thinking block logic:** Refined telegram thinking block streaming and delivery logic to fix odd spacing/newlines.

### Changed
- **Notion plugin collapsed to a single `ntn` tool and relocated to plugins folder.** The 18 hand-rolled wrappers (`notion_search`, `notion_read`, `notion_create`, `notion_doctor`, `notion_logs_read`, …) are replaced by one `ntn` tool that proxies arbitrary args to the official Notion CLI, reducing tool parameter bloat. The Notion integration has been relocated from `backend/app/integrations/notion/` to `backend/app/plugins/notion/`, breaking static import cycles with the Telegram bot. Token isolation, ephemeral `HOME`, and `notion_operation_logs` auditing remain in place.
- **Improved Notion tool rendering and display formatting on Web and Telegram.** Added specific formatting and icons for common Notion commands, including:
  - Notion diagnostics (`ntn doctor` → `🩺 Running Notion diagnostics...` / `Ran Notion diagnostics`)
  - Notion pages listing (`ntn pages list` → `📖 Listing Notion pages...` / `Listed Notion pages`)
  - Notion API endpoint listing (`ntn api ls` → `📋 Listing Notion API endpoints...` / `Listed Notion API endpoints`)
  - Database queries via raw API (`ntn api v1/databases/<id>/query` → `🔍 Querying Notion database <id_truncated>...` / `Queried Notion database <id_truncated>`)
  - Database schema retrievals via raw API (`ntn api v1/databases/<id>` → `🗂️ Reading Notion database schema <id_truncated>...` / `Read Notion database schema <id_truncated>`)
- **Truncation of page and database UUIDs.** Display labels for database queries and schema reads now truncate UUIDs to their first 8 characters to keep user-facing text clean.
- **Tightened `ntn` tool arguments validation.** The `ntn` tool now rejects non-string arguments explicitly, and its description has been clarified to ensure the agent uses correct formatting.
- **Frontend dev port swap:** Swapped the frontend dev port from 3001 to 53001.
- **Deps bump:** Bumped GitHub actions dependencies (#465).
- **Testing:** Pinned must-have model ids for the catalog test (#385), closed out integration test gaps (#393).

- **Telegram voice notes are now transcribed under the default xAI deployment.** Previously the Telegram bot only attempted transcription when `VOICE_PROVIDER` was set to `mistral`, `openai`, or `local`; the default `xai` value made `resolve_transcriber()` return `None` and the bot fell back to the "Transcription is not available on this surface yet" stub annotation, so the agent never saw the spoken content. A new `XaiSttTranscriber` wraps xAI's HTTP STT endpoint (`https://api.x.ai/v1/stt`) and is now returned by `resolve_transcriber()` whenever `voice_provider == "xai"` and `settings.xai_api_key` is set. The web `POST /api/v1/stt` route was refactored to use the same transcriber class with workspace-first `XAI_API_KEY` resolution — same observable behaviour for web users, less duplicated code. Image and document attachment paths were diagnosed at the same time: images already forward correctly through `ChatTurnInput.images` for Claude (other providers still log-and-drop, tracked as a follow-up); documents already extract via markitdown when the optional dep is installed.

The catalog's `supports_reasoning` tuples advertise the levels each model actually honours so the picker can't surface a level the provider would silently drop.
- **Gemini cost tracking + per-model thinking defaults** (#362) — Gemini previously emitted no token / cost events; every turn landed in the cost ledger with zeroes. Thinking tokens are billed as output per the Gemini Thinking docs. The per-turn `thinking_level` is now pinned to a per-model safe default when no override is set (Flash-Lite → `minimal`, 3-Flash / 3.5-Flash → `low`, 3.1-Pro → `medium`) instead of Gemini 3's upstream `high` default — which counts thinking tokens against `maxOutputTokens` and routinely exhausts the output budget on tool follow-up turns, manifesting as the `TELEGRAM_TOOL_ONLY_TURN` empty-bubble bug (see openclaw/openclaw#30616).

- **Claude conversation context preserved on provider switches** (#308) — when a user switches from Gemini (or any non-Claude provider) to Claude mid-conversation, the Claude SDK has no transcript for the `conversation_id` and `history` was silently ignored. The provider now renders the app-level history (already loaded by the chat router from `chat_messages`) as a bounded `--- BEGIN PRIOR CONTEXT ---` block prepended to the user's question, so the model sees the prior turns instead of starting blind. When Claude already has its own transcript (`resume=` works), the prefix is skipped to avoid duplicating context.
- **Python virtual tool now shows the actual code** in tool-trace surfaces. Previously the Telegram + web chain-of-thought rendered only `(code)` (the argument key, not the value). Now surfaces the first non-blank line in `present` / `compact` and the full source (head-truncated, fenced) in `detail`. (#302)
- **TelegramConnectDialog** swapped from raw shadcn `Dialog` to the project's `AppDialog` (responsive modal + bottom-sheet primitive); replaced the `(_, next) => next` useReducer with `useState`; and fixed every `rel="noreferrer"` anchor to the canonical `rel="noopener noreferrer"`. (#270 — partial; view/container split + custom Biome rule deferred to a sibling PR)
- **Telegram thinking text now renders Markdown emphasis** instead of leaking literal `**` / `_` markers into the italic block. `thinking_html` routes the text through the same `md_to_telegram_html` pipeline the answer stream uses, then wraps the result in `<i>...</i>` (Telegram allows `<b>` / `<a>` nested inside `<i>`). (#287)
- **Tool-only Telegram turns no longer dead-end silently** — when a turn calls tools but finishes without any answer or thinking text, the channel now sends the empty-response fallback as a closing message instead of leaving the user staring at the tool trace and silence. A new `TELEGRAM_TOOL_ONLY_TURN` warning fires in the log so it's diagnosable. (#293)
- **Frontend test isolation** — `frontend/test/setup.ts` now clears the in-memory `localStorage` polyfill on `afterEach` so persisted state (selected model, sidebar collapse, plan-mode flags, …) from one test no longer leaks into the next. The chat router, Telegram bot, and event-bus webhook handlers all forward `workspace_root`. (#259)

- **Railway Docker build crashing on `ntn` install** — the upstream installer (`curl https://ntn.dev | bash`) writes directly to `/usr/local/bin/ntn` when run as root, so the follow-up `mv /root/.ntn/bin/ntn /usr/local/bin/ntn` failed with "No such file or directory" and aborted the build. Installed `libpq5` in the runtime apt step. (#299)
