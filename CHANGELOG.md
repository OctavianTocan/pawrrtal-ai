# Changelog

## Unreleased

### Added
- **External MCP server invocation** (#317) — agents can now call into user-configured external MCP servers as cross-provider `AgentTool`s. Storage lives in the new `mcp_servers` table (migration `017`, model in `app/mcp_models.py`); the per-user CRUD API at `/api/v1/mcp/servers` supports `GET`/`POST`/`PATCH`/`DELETE`. Discovery hits `POST <url>/list_tools`, invocation hits `POST <url>/call_tool`; both are timeout-capped (30s) and response-bounded (32 kB). Tools are name-prefixed `mcp_<server>_<tool>` to avoid collisions and additive at the tail of the agent tool list so a slow remote server can never destabilise the core surface.
- **Telegram voice transcription wired into attachment processing** (#304) — voice + audio messages now travel through `resolve_transcriber()` (Mistral / OpenAI / local whisper.cpp) inside `collect_attachments`, so the agent sees `[User sent a voice message (Ns). Transcription: ...]` instead of a metadata-only stub. Failed transcription falls back to the bounded annotation; voice ingestion never breaks a turn.
- **Telegram document extraction via markitdown-on-bytes** (#305) — `_attachments.py` writes the downloaded document to a private tempdir (preserving the extension so markitdown picks the right reader), runs the synchronous `MarkItDown` call in a worker thread, and inlines a bounded `[User sent a document: ... Extracted Markdown: ...]` excerpt (capped at 8,000 chars). Oversized or unsupported documents fall back to a metadata annotation.
- **Chronological text-block rendering on Telegram** (#306, #307) — interleaved text deltas that follow a thinking or tool block now open their own Telegram message in the chronological position the model actually emitted them, instead of being batched into a single trailing reply. Pure-text turns keep the original single-message UX. `_telegram_dispatch.py` grows `prepare_text_block` / `handle_text_delta` helpers mirroring the thinking/tools path.
- **Skill discovery + invocation agent tools** (#315) — three new `AgentTool`s (`list_skills`, `read_skill`, `invoke_skill`) wrap the workspace skill manifest reader. `invoke_skill` returns the bounded SKILL.md body wrapped in clear BEGIN/END markers so the agent treats it as the playbook for the next steps. Hard caps: 800 lines / 32 kB per skill body; path traversal rejected with `ToolError(OUT_OF_ROOT)`.
- **Telegram inbound attachment processing** (`backend/app/integrations/telegram/_attachments.py`) — the bot no longer silently drops messages with attachments. Photo attachments are downloaded, base64-encoded, and forwarded as `image_inputs` through `ChatTurnInput.images` so vision-capable models can read them. (Earlier scaffolding; #304/#305 close the voice + document extensions described above.)
- **Biome `noRestrictedImports` rule** banning raw shadcn `@/components/ui/{dialog,alert-dialog,sheet}` imports from `frontend/features/**`. The existing `frontend/features/onboarding/**` flow is carved out as known tech debt — its nested DialogHeader/DialogDescription usage predates `AppDialog`. (#270 part 2)
- **Telegram block-transition rendering** (#288) — alternating `thinking → tools → thinking` sequences now produce three separate Telegram messages instead of two ever-growing blobs. The channel tracks `previous_block_kind` and opens a fresh Telegram message on every kind change; the placeholder is consumed by the *first* block whichever it is.
- **`@axe-core/playwright` accessibility smoke specs** (#276) — four specs under `frontend/e2e/a11y.spec.ts` cover login, authenticated home shell, settings, and chat composer with a draft. WCAG 2.0 + 2.1, levels A + AA.
- **Single-alembic-head CI gate** (#296) — Backend Check now fails when `alembic heads` reports more than one head, catching the multi-head class of bug that motivated #295 at PR time instead of deploy time.
- **Per-turn runtime context block** (`backend/app/core/runtime_context.py`) appended to the agent system prompt every turn. Surfaces the current UTC time and weekday so the model never burns iterations on `exa_search` queries for the date (#294); the active provider + model id + display name so "which model are you?" lands trusted runtime metadata instead of a training-data guess (#309); the per-turn `max_iterations` / `max_wall_clock_seconds` from the safety layer so the model can plan a finishable turn (#291); and a one-line inventory of every `AgentTool` bound for the turn so the agent stops scanning the workspace to discover its own capabilities (#289). Composition lives in `backend/app/channels/_turn_runtime_context.py` to keep `turn_runner.py` under sentrux's fan-out budget. (#289, #291, #294, #309)
- **`now()` agent tool** (`backend/app/core/tools/now.py`) — pure stdlib, zero network, always available. Returns ISO-8601 UTC + local time + IANA timezone + day-of-week. Lets long, multi-step turns re-query the clock without burning an iteration on a web search. (#294)
- **`/lcm` Telegram slash command** (`backend/app/integrations/telegram/lcm_status.py`) — diagnostic surface for Lossless Context Management. Reports whether `settings.lcm_enabled` is on, the context-item composition (raw vs compacted), summary breakdown by depth + kind, and the most recent summary's age + tokens. Disabled-LCM gets its own static reply so the operator can spot silent off-by-default deploys. (#303)
- **Playwright E2E in CI** (`.github/workflows/playwright-e2e.yml`) — runs `just e2e` on ephemeral backend + frontend dev servers against every PR. Gated on the OctavianTocan actor + self-hosted runner per project CI rules; uploads the Playwright HTML report + per-test results as failure artifacts. (#273)
- **`SQLITE_DB_FILENAME` env var** in `Settings` — lets parallel checkouts / experiments point the implicit SQLite default at a different `.db` file without spelling out a full `sqlite:///...` URL. An explicit `DATABASE_URL` always wins. (#279)
- **`test:coverage` script + CI artifact upload** — Vitest now runs with v8 coverage in CI and uploads `frontend/coverage/` as a downloadable artifact. (#274)
- **`just test` runs both suites** — `test-backend`, `test-frontend`, and `test-coverage` sub-recipes added so each suite can be invoked independently. (#271)
- **Python module-privacy rule** (`.claude/rules/clean-code/python-module-privacy.md`) — codifies the leading-underscore convention. Ruff `PLC2701` opt-in path documented for the follow-up sweep that will mechanically enforce it. (#283)
- **Python parameter-order rule** (`.claude/rules/clean-code/python-parameter-order.md`) — documents canonical session-first / identity-second order for backend CRUD + service helpers. (#282)
- **Plugin system** (`backend/app/core/plugins/`) — additive integrations declare env keys + tool factories; `build_agent_tools` walks the registry as a tail-append after the core tools. Existing tools are unchanged. (#261)
- **Notion integration** as the first plugin — 18 tools mirroring `openclaw-notion`'s contract (`notion_search`, `notion_read`, `notion_create`, `notion_update_markdown`, `notion_query`, `notion_sync`, …). Executes via per-call `ntn` subprocess with `NOTION_API_TOKEN` injected and `HOME` isolated to a tempdir; every call is captured to a new `notion_operation_logs` table (alembic `013`) that `notion_logs_read` surfaces back to the agent. (#261)
- **Workspace-keyed encrypted `.env`** — credentials move from per-user (`{base}/{user_id}/.env`) to per-workspace (`{base}/{workspace_id}/.env`). `resolve_api_key(workspace_id, key)` replaces the user-keyed signature across providers, tools, STT, and the chat router; the env API is now `/api/v1/workspaces/{workspace_id}/env`. A non-destructive startup migration moves legacy files into each user's default workspace (source renamed `.env.migrated-<ts>`). Operators get a WARN log per user with secondary workspaces so they know those start with an empty credentials slate. (#261)
- **In-process `python` agent tool** (off by default, gated by `settings.virtual_python_enabled`) — runs LLM-supplied Python via `exec()` in the FastAPI worker with stdout+stderr capture, head/tail truncation, a wall-clock timeout, and a workspace-jailed `fs` global. Calls serialise via a module-level `asyncio.Lock`; process state (`sys.path`, `os.environ`, `warnings.filters`, root logger) is snapshotted and restored per call. Trust model: single-tenant self-hosted only — code can reach `os.environ`, `subprocess`, sockets. (#266)
- **Shared tool display metadata** — Telegram tool-use messages now show compact friendly text instead of raw JSON argument dumps; the web chain-of-thought renders the same friendly form. Provider-agnostic — formatters live in `backend/app/core/tools/display.py` and never reach the provider layer. (#262)
- **Telegram `/status` command** — gateway uptime + model + verbose level + per-conversation message + token counts + run state. (#262)
- **Live Telegram tool trace** — tool calls collect into one edited message with icons, argument-key previews, and JSON inputs; thinking and final replies stay in their own messages.
- **Telegram model browsing** — `/models` and `/model list` open an inline-keyboard picker instead of requiring the user to type a canonical model ID.
- **First-run Paw persona bootstrap** — each assistant knows it is the user's Paw and can ask what name and personality it should grow into.
- **Lossless Context Management (LCM)** — DAG-based conversation compaction. Older turns are summarised into queryable `LCMSummary` nodes while a verbatim fresh tail is preserved. Off by default behind `lcm_enabled`; opt-in via six settings (`lcm_fresh_tail_count`, `lcm_leaf_chunk_tokens`, `lcm_context_threshold`, `lcm_incremental_max_depth`, `lcm_summary_model`).
- **Four LCM agent tools** — `lcm_grep` / `lcm_describe` / `lcm_list_summaries` / `lcm_expand_query` so the agent can search, enumerate, inspect, and synthesise across compacted history when context grows past the fresh tail.
- **Background leaf compaction** after every chat turn, plus a condensation pass that folds same-depth summaries into deeper parent nodes (configurable up to an unlimited cascade).
- **Provider-native replay state** — the agent loop forwards opaque `provider_state` from `LLMDoneEvent` onto the assistant message so Gemini can replay `ModelContent` (preserving `thought_signature` bytes) on follow-up tool turns without leaking the slot into transcripts or persistence.
- **Migrations** — `015_add_lcm_tables` (`lcm_summaries`, `lcm_summary_sources`, `lcm_context_items`), and `016_merge_notion_into_lcm_lineage` (alembic merge that joins the Notion + LCM lineages).
- **Startup refresh for Telegram slash commands** so the bot menu matches the current server build.
- **Verbose Telegram delivery** for tool activity and thinking events.

### Changed
- **AGENTS.md template** (#290) — replaced the "On every session start, read…" block with "Session Continuity" guidance that tells the Paw the relevant files are already loaded into the system prompt for this turn. New workspaces seeded after this change get the new wording; existing workspaces keep their content. Stops the per-turn read ritual that burned 4-6 tool calls on context the runtime already supplies.
- **Reply-to wiring on Telegram** — thinking, tool traces, and final answers reply to the user's original message when Telegram gives us a message id.
- **Agent tool labels** renamed from `ai_nexus` → `pawrrtal` (and every other reference; see below).
- **Repo-wide rename** `AI Nexus` / `nexus` → `Pawrrtal` / `pawrrtal` across 162 files: code identifiers (`nexus_user_id` → `pawrrtal_user_id`, `X-Nexus-Surface` → `X-Pawrrtal-Surface`), brand strings, GitHub URLs (`OctavianTocan/ai-nexus` → `OctavianTocan/Pawrrtal-AI`), env defaults (`nexus-ai.dev` → `pawrrtal-ai.dev`), SVG `aria-label`s, 19 leftover `.beans/ai-nexus-*` files that lacked a `pawrrtal-*` twin, and the `docs/hyperframes/ai-nexus-demo-reel/` directory. Vendored submodules under `frontend/lib/{react-chat-composer,react-dropdown,react-overlay}/` and alembic revision IDs are intentionally untouched. (#292)
- **Auto Claude Code Review CI workflow removed** — it hung repeatedly and blocked the single-runner queue. The explicit `@claude`-mention workflow (`claude.yml`) stays. (#284)

### Fixed
- **Claude conversation context preserved on provider switches** (#308) — when a user switches from Gemini (or any non-Claude provider) to Claude mid-conversation, the Claude SDK has no transcript for the `conversation_id` and `history` was silently ignored. The provider now renders the app-level history (already loaded by the chat router from `chat_messages`) as a bounded `--- BEGIN PRIOR CONTEXT ---` block prepended to the user's question, so the model sees the prior turns instead of starting blind. When Claude already has its own transcript (`resume=` works), the prefix is skipped to avoid duplicating context.
- **Python virtual tool now shows the actual code** in tool-trace surfaces. Previously the Telegram + web chain-of-thought rendered only `(code)` (the argument key, not the value). Now surfaces the first non-blank line in `present` / `compact` and the full source (head-truncated, fenced) in `detail`. (#302)
- **TelegramConnectDialog** swapped from raw shadcn `Dialog` to the project's `AppDialog` (responsive modal + bottom-sheet primitive); replaced the `(_, next) => next` useReducer with `useState`; and fixed every `rel="noreferrer"` anchor to the canonical `rel="noopener noreferrer"`. (#270 — partial; view/container split + custom Biome rule deferred to a sibling PR)
- **Telegram thinking text now renders Markdown emphasis** instead of leaking literal `**` / `_` markers into the italic block. `thinking_html` routes the text through the same `md_to_telegram_html` pipeline the answer stream uses, then wraps the result in `<i>...</i>` (Telegram allows `<b>` / `<a>` nested inside `<i>`). (#287)
- **Tool-only Telegram turns no longer dead-end silently** — when a turn calls tools but finishes without any answer or thinking text, the channel now sends the empty-response fallback as a closing message instead of leaving the user staring at the tool trace and silence. A new `TELEGRAM_TOOL_ONLY_TURN` warning fires in the log so it's diagnosable. (#293)
- **Frontend test isolation** — `frontend/test/setup.ts` now clears the in-memory `localStorage` polyfill on `afterEach` so persisted state (selected model, sidebar collapse, plan-mode flags, …) from one test no longer leaks into the next. Vitest's `clearMocks: true` is also enabled so spy-call assertions can't accidentally share state across tests. (#278)
- **`seeded_default_workspace` test fixture no longer leaks temp dirs** — switched from `tempfile.mkdtemp` (no cleanup) to pytest's `tmp_path` so the workspace root is removed after every test, fixing the slow accumulation of `ainexus-test-ws-*` directories under `$TMPDIR`. (#275)
- **Frontend Check CI workflow now actually gates Biome config changes** — `.github/workflows/check.yml`'s `paths:` filter referenced `biome.jsonc` (which doesn't exist); the real file is `biome.json`. PRs that only touched `biome.json` were skipping the gate they were meant to control. (#272)
- **Workspace-isolation gap in the Claude Agent SDK** — every chat turn was reading the backend repo's `CLAUDE.md`, `.claude/settings.json` (hooks!), and `.mcp.json` from the uvicorn cwd. Pinned `setting_sources=[]` unconditionally and plumbed per-user `workspace_root` into `ClaudeLLMConfig.cwd` so the SDK subprocess's transcript directory lands under the user workspace. The chat router, Telegram bot, and event-bus webhook handlers all forward `workspace_root`. (#259)
- **Backend-wide mypy zero-error sweep** — cleared 60 strict-mypy errors across 29 files (now 0 errors across 138 source files). Typing-only sweep — no runtime behaviour change. Caught one real latent bug: `middleware.py` had three `# type: ignore[misc]` masking a wrong `call_next: object` annotation; replaced with `RequestResponseEndpoint` and dropped the ignores. (#259)
- **Gemini `system_prompt` actually reaching the SDK** — `make_gemini_stream_fn` was hardcoding the module-level fallback on every `GenerateContentConfig.system_instruction`, ignoring the per-request prompt the chat router passed via `provider.stream(system_prompt=...)`. The workspace-assembled prompt (`SOUL.md` + `AGENTS.md` + `CLAUDE.md` + skills) now actually reaches Gemini. (#264)
- **Two workspace-jail vulnerabilities** in `_resolve_safe`: sibling-prefix bypass (`/data/workspaces/abc-evil` against `/data/workspaces/abc`) via switch from `str.startswith` to `Path.is_relative_to`; and read/write symlink TOCTOU via `O_NOFOLLOW` on every open (`read_file`, `write_file`, and the `python` tool's `WorkspaceFS.{read,read_bytes,write,write_bytes,append}`). `fs.glob` filters symlinks. (#266)
- **Railway deploy port binding** — wrapping `startCommand` in `sh -c` so `$PORT` expands at runtime instead of reaching uvicorn as a literal string; uvicorn now `exec`'d so SIGTERM reaches it directly. (#269)
- **Alembic multi-head defect** — reparented `001_add_conversation_metadata`'s `down_revision` from `None` onto `000_initial_schema`, and added `016_merge_notion_into_lcm_lineage` joining `013_notion_operation_logs` + `015_add_lcm_tables`. `alembic heads` now reports a single head; rolling-deploy concurrency is serialised by a Postgres advisory lock around `context.run_migrations()`. (#286, #295)
- **Railway Docker build crashing on `ntn` install** — the upstream installer (`curl https://ntn.dev | bash`) writes directly to `/usr/local/bin/ntn` when run as root, so the follow-up `mv /root/.ntn/bin/ntn /usr/local/bin/ntn` failed with "No such file or directory" and aborted the build. Dropped the redundant `mv`. (#295)
- **Railway boot crashing on `psycopg` import** — `backend/pyproject.toml` depends on plain `psycopg` (no `[binary]` extra), which loads `libpq` via ctypes at import time. `python:3.13-slim` doesn't ship `libpq`, so the runtime image's first `from app import models` (triggered by `alembic upgrade head` in `startCommand`) raised `ImportError: no pq wrapper available`. Installed `libpq5` in the runtime apt step. (#299)
- **Default workspace creation races** — losing inserts clean up their seeded directory asynchronously.
