- Added a **plugin system** (`backend/app/core/plugins/`) â€” additive integrations declare env keys + tool factories and `build_agent_tools` walks the registry as a tail-append after core tools. Existing tools are unchanged. (#261)
- Added a **Notion integration** as the first plugin: 18 tools mirroring `openclaw-notion`'s contract (`notion_search`, `notion_read`, `notion_create`, `notion_update_markdown`, `notion_query`, `notion_sync`, â€¦). - Migrated **workspace `.env` storage from per-user to per-workspace** (`{base}/{workspace_id}/.env`). `resolve_api_key(workspace_id, key)` replaces the user-keyed signature across providers, tools, STT, and the chat router; the env API is now `/api/v1/workspaces/{workspace_id}/env`. Operators get a WARN log per user with secondary workspaces so they know those start with an empty credentials slate. (#261)
- Added an **in-process `python` agent tool** (off by default, gated by `settings.virtual_python_enabled`). Runs LLM-supplied Python via `exec()` in the FastAPI worker with stdout+stderr capture, head/tail truncation, a wall-clock timeout, and a workspace-jailed `fs` global. Calls serialise via a module-level asyncio lock; process state (`sys.path`, `os.environ`, `warnings.filters`, root logger) is snapshotted and restored per call. Trust model: single-tenant self-hosted only â€” code can reach `os.environ`, `subprocess`, sockets. (#266)

- Added a Telegram **`/status` command** that shows worker uptime, model + verbose level, conversation message + token counts, and live run state.
- Added a live Telegram tool trace: tool calls now collect into one edited message with icons, argument-key previews, and JSON inputs, while thinking text and final replies stay in their own messages.

- Added Lossless Context Management (LCM): a DAG-based conversation compaction system that summarises older turns into queryable `LCMSummary` nodes while preserving a verbatim fresh tail. Off by default behind `lcm_enabled`; opt-in via six new settings (`lcm_fresh_tail_count`, `lcm_leaf_chunk_tokens`, `lcm_context_threshold`, `lcm_incremental_max_depth`, `lcm_summary_model`).
- Added four LCM agent tools (`lcm_grep`, `lcm_describe`, `lcm_list_summaries`, `lcm_expand_query`) so the agent can search, enumerate, inspect, and synthesise across compacted history when context grows past the fresh tail.
- Added background leaf compaction after every chat turn, plus a condensation pass that folds same-depth summaries into deeper parent nodes (configurable up to an unlimited cascade).
- Added provider-native replay state: the agent loop now forwards opaque `provider_state` from `LLMDoneEvent` onto the assistant message so Gemini can replay `ModelContent` (preserving `thought_signature` bytes) on follow-up tool turns without leaking the slot into transcripts or persistence.
