# Changelog

## Unreleased

### Added
- **Plugin system** (`backend/app/core/plugins/`) — additive integrations declare env keys + tool factories; `build_agent_tools` walks the registry as a tail-append after the core tools. Existing tools are unchanged. (#261)
- **Notion integration** as the first plugin — 18 tools mirroring `openclaw-notion`'s contract (`notion_search`, `notion_read`, `notion_create`, `notion_update_markdown`, `notion_query`, `notion_sync`, …). Executes via per-call `ntn` subprocess with `NOTION_API_TOKEN` injected and `HOME` isolated to a tempdir; every call is captured to a new `notion_operation_logs` table (alembic `013`) that `notion_logs_read` surfaces back to the agent. (#261)
- **Workspace-keyed encrypted `.env`** — credentials move from per-user (`{base}/{user_id}/.env`) to per-workspace (`{base}/{workspace_id}/.env`). `resolve_api_key(workspace_id, key)` replaces the user-keyed signature across providers, tools, STT, and the chat router; the env API is now `/api/v1/workspaces/{workspace_id}/env`. A non-destructive startup migration moves legacy files into each user's default workspace (source renamed `.env.migrated-<ts>`). Operators get a WARN log per user with secondary workspaces so they know those start with an empty credentials slate. (#261)
- **In-process `python` agent tool** (off by default, gated by `settings.virtual_python_enabled`) — runs LLM-supplied Python via `exec()` in the FastAPI worker with stdout+stderr capture, head/tail truncation, a wall-clock timeout, and a workspace-jailed `fs` global. Calls serialise via a module-level `asyncio.Lock`; process state (`sys.path`, `os.environ`, `warnings.filters`, root logger) is snapshotted and restored per call. Trust model: single-tenant self-hosted only — code can reach `os.environ`, `subprocess`, sockets. (#266)
- **Shared tool display metadata** — Telegram tool-use messages now show compact friendly text instead of raw JSON argument dumps; the web chain-of-thought renders the same friendly form. Provider-agnostic — formatters live in `backend/app/core/tools/display.py` and never reach the provider layer. (#262)
- **Telegram `/status` command** — gateway uptime + model + verbose level + per-conversation message + token counts + run state. (#262)
- **Live Telegram tool trace** — tool calls collect into one edited message with icons, argument-key previews, and JSON inputs; thinking and final replies stay in their own messages.
- **Telegram model browsing** — `/models` and `/model list` open an inline-keyboard picker instead of requiring the user to type a canonical model ID.
- **First-run Paw persona bootstrap** — each assistant knows it is the user's Paw and can ask what name and personality it should grow into.
- **Lossless Context Management (LCM)** — DAG-based conversation compaction. Older turns are summarised into queryable `LCMSummary` nodes while a verbatim fresh tail is preserved. Off by default behind `lcm_enabled`; opt-in via six settings (`lcm_fresh_tail_count`, `lcm_leaf_chunk_tokens`, `lcm_context_threshold`, `lcm_incremental_max_depth`, `lcm_summary_model`).
- **Four LCM agent tools** — `lcm_grep` / `lcm_describe` / `lcm_list_summaries` / `lcm_expand_query` so the agent can search, enumerate, inspect, and synthesise across compacted history when context grows past the fresh tail.
- **Background leaf compaction** after every chat turn, plus a condensation pass that folds same-depth summaries into deeper parent nodes (configurable up to an unlimited cascade).
- **Provider-native replay state** — the agent loop forwards opaque `provider_state` from `LLMDoneEvent` onto the assistant message so Gemini can replay `ModelContent` (preserving `thought_signature` bytes) on follow-up tool turns without leaking the slot into transcripts or persistence.
- **Migrations** — `015_add_lcm_tables` (`lcm_summaries`, `lcm_summary_sources`, `lcm_context_items`), and `016_merge_notion_into_lcm_lineage` (alembic merge that joins the Notion + LCM lineages).
- **Startup refresh for Telegram slash commands** so the bot menu matches the current server build.
- **Verbose Telegram delivery** for tool activity and thinking events.

### Changed
- **Reply-to wiring on Telegram** — thinking, tool traces, and final answers reply to the user's original message when Telegram gives us a message id.
- **Agent tool labels** renamed from `ai_nexus` → `pawrrtal` (and every other reference; see below).
- **Repo-wide rename** `AI Nexus` / `nexus` → `Pawrrtal` / `pawrrtal` across 162 files: code identifiers (`nexus_user_id` → `pawrrtal_user_id`, `X-Nexus-Surface` → `X-Pawrrtal-Surface`), brand strings, GitHub URLs (`OctavianTocan/ai-nexus` → `OctavianTocan/Pawrrtal-AI`), env defaults (`nexus-ai.dev` → `pawrrtal-ai.dev`), SVG `aria-label`s, 19 leftover `.beans/ai-nexus-*` files that lacked a `pawrrtal-*` twin, and the `docs/hyperframes/ai-nexus-demo-reel/` directory. Vendored submodules under `frontend/lib/{react-chat-composer,react-dropdown,react-overlay}/` and alembic revision IDs are intentionally untouched. (#292)
- **Auto Claude Code Review CI workflow removed** — it hung repeatedly and blocked the single-runner queue. The explicit `@claude`-mention workflow (`claude.yml`) stays. (#284)

### Fixed
- **Workspace-isolation gap in the Claude Agent SDK** — every chat turn was reading the backend repo's `CLAUDE.md`, `.claude/settings.json` (hooks!), and `.mcp.json` from the uvicorn cwd. Pinned `setting_sources=[]` unconditionally and plumbed per-user `workspace_root` into `ClaudeLLMConfig.cwd` so the SDK subprocess's transcript directory lands under the user workspace. The chat router, Telegram bot, and event-bus webhook handlers all forward `workspace_root`. (#259)
- **Backend-wide mypy zero-error sweep** — cleared 60 strict-mypy errors across 29 files (now 0 errors across 138 source files). Typing-only sweep — no runtime behaviour change. Caught one real latent bug: `middleware.py` had three `# type: ignore[misc]` masking a wrong `call_next: object` annotation; replaced with `RequestResponseEndpoint` and dropped the ignores. (#259)
- **Gemini `system_prompt` actually reaching the SDK** — `make_gemini_stream_fn` was hardcoding the module-level fallback on every `GenerateContentConfig.system_instruction`, ignoring the per-request prompt the chat router passed via `provider.stream(system_prompt=...)`. The workspace-assembled prompt (`SOUL.md` + `AGENTS.md` + `CLAUDE.md` + skills) now actually reaches Gemini. (#264)
- **Two workspace-jail vulnerabilities** in `_resolve_safe`: sibling-prefix bypass (`/data/workspaces/abc-evil` against `/data/workspaces/abc`) via switch from `str.startswith` to `Path.is_relative_to`; and read/write symlink TOCTOU via `O_NOFOLLOW` on every open (`read_file`, `write_file`, and the `python` tool's `WorkspaceFS.{read,read_bytes,write,write_bytes,append}`). `fs.glob` filters symlinks. (#266)
- **Railway deploy port binding** — wrapping `startCommand` in `sh -c` so `$PORT` expands at runtime instead of reaching uvicorn as a literal string; uvicorn now `exec`'d so SIGTERM reaches it directly. (#269)
- **Alembic multi-head defect** — reparented `001_add_conversation_metadata`'s `down_revision` from `None` onto `000_initial_schema`, and added `016_merge_notion_into_lcm_lineage` joining `013_notion_operation_logs` + `015_add_lcm_tables`. `alembic heads` now reports a single head; rolling-deploy concurrency is serialised by a Postgres advisory lock around `context.run_migrations()`. (#286, #295)
- **Railway Docker build crashing on `ntn` install** — the upstream installer (`curl https://ntn.dev | bash`) writes directly to `/usr/local/bin/ntn` when run as root, so the follow-up `mv /root/.ntn/bin/ntn /usr/local/bin/ntn` failed with "No such file or directory" and aborted the build. Dropped the redundant `mv`. (#295)
- **Default workspace creation races** — losing inserts clean up their seeded directory asynchronously.
