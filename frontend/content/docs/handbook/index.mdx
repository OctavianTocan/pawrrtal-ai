# Pawrrtal Handbook

This is the durable reference set for Pawrrtal — the bits worth
writing down once and pointing at later. Working notes (in-flight
plans, debug write-ups) stay as raw markdown in the repo and are
intentionally not rendered here.

## Mental model

Pawrrtal is a multi-provider AI chat product organized around four
layers:

1. **Frontend** — Next.js 16 App Router + React 19 + Tailwind v4.
   Talks to the backend through `useAuthedFetch` / TanStack Query
   only. Lives in `frontend/`.
2. **Backend** — FastAPI on Python 3.13, SQLAlchemy 2 async +
   Alembic, fastapi-users for JWT cookie auth. Lives in `backend/`.
3. **Agent loop** — provider-agnostic event loop (`StreamFn` seam +
   `AgentTool[]`) with two providers wired in today: Claude (via
   `claude-agent-sdk`) and Gemini (via `google-genai`). The seam is
   what `ScriptedStreamFn` tests target.
4. **Channels** — surfaces a conversation can arrive through. Web is
   SSE; Telegram is `aiogram` with both polling and webhook modes.

The plugin system layers on top: a `Plugin` provides one or more
`AgentTool`s and is opted into per workspace. Notion is the
reference plugin (18 tools, all shelling out to `ntn`).

LCM (Lossless Context Management) is the conversation-compaction
DAG that lets long sessions survive without losing causal links.

```
backend/
  app/
    api/                 FastAPI routes (one file per domain)
    channels/            web SSE + Telegram adapters
    core/
      agent_loop/        provider-agnostic loop + StreamFn seam
      providers/         claude_provider, gemini_provider
      tools/             AgentTool factories
    integrations/        plugins, including notion/

    models.py            SQLAlchemy ORM
  alembic/versions/      migration history (single head as of 016)
frontend/
  app/                   Next.js App Router routes
  features/              feature modules (container + view + hooks)
  components/            shared UI (shadcn-style + Craft tokens)
  content/docs/          this handbook (rendered by Fumadocs)

.beans/                  markdown task tracking
.claude/rules/           path-scoped rules for agents
DESIGN.md                canonical design tokens (mirrors globals.css)
```

## Conventions worth knowing before your first PR

- **`just` is the task runner.** `just dev` for both servers,
  `just check` for the unified lint gate, `just check-all` for the
  full pre-push gate.
- **500-line file cap** on `.ts` / `.tsx` / `.py` (enforced by
  `scripts/check-file-lines.mjs`).
- **Nesting depth ≤ 3** in functions
  (`scripts/check-nesting.{py,mjs}`).
- **No `any`, no `@ts-nocheck`, no Tailwind literal colours.** Use
  the design tokens.
- **Agent-loop tests use `ScriptedStreamFn`**, not patched
  `agent_loop`. See `backend/tests/agent_harness.py` and the
  testing rules.
- **Conventional Commits.** PRs land small and reviewable.
